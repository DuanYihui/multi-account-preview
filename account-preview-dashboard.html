多套账号预览工作台

每个环境账号共用一个独立 Chrome profile。同一账号里的移动端和 Web 端共享 Cookie；不同账号之间互相隔离。

最短启动命令 tools/multi-account-preview/preview-accounts.sh init tools/multi-account-preview/preview-accounts.sh start

怎么用

先执行 init，它会生成账号菜单：
~/.account-preview-workbench/accounts.tsv

把环境、账号、页面名、页面地址和窗口位置填进去。同一个环境账号会共用同一份 Cookie。

执行 start。第一次在某个账号的移动端登录后，同账号 Web 端就能按浏览器规则读取可共享的 Cookie。

账号菜单预览

这里可以先排一下窗口。最后一列 profileKey 是账号内部编号，账号改名时不要跟着改它，这样登录态不会丢。
